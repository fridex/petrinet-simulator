Introduction

PN2012 is a Petri net simulator and editor. It was developed as a school project at Faculty of Information Technology, University of Technology in Brno. Application consists of two parts - a server and a client-side editor. Editor is used to edit and create new Petri nets. Petri nets can be simulated on server, which can be used as a remote repository for storing various projects in different versions created by registered users.

Connecting to server

If you want to store your net in a remote repository or you want to simulate your Petri net, you have to be connected to the server.

To connect to the remote server, select Repository from menu bar and then click on Connect. A new window with connection information will appear. Specify Server field and edit Port number to connect to. Then set up your login information to authorise yourself on server-side. If you want to create new account, check Register new user.

If authorisation or registration was successful, you will be able to simulate, view simulation log, save your project in repository or download projects from repository.

Creating and editing Petri net

You can select Arrow (arc), Place or Transition from toolbox to be placed to an area. Once you place selected item to a scene, you can move item by selecting item and holding left mouse button. Arrows connected to the moved item will be automatically adjusted. You can remove item from scene as well, by selecting Remove item icon from toolbar and clicking on item you want to remove. Arrows will be deleted automatically to respect net consistency.

To adjust or enter values for an item select Edit icon from toolbar and then select the item you want to edit. Based on the item you selected, an edit window will appear to adjust item information. If you select a place, you can adjust place name and tokens, which are stored in the place. On the other hand, if you select a transition, you can adjust a condition which has to be true for tokens to pass the transition. Setting up mode you specify output of the transition. You can edit name of the selected transition as well. If you select an arrow, you can edit arrow name.

Petri net can be saved using File - Save or File - Save as mechanism. Specify a file location and a file name to save the net. Once you save your net, application will remember the location and you can use just CTRL + S key binding to overwrite older version. If you want to load previously stored net, select File - Open and Petri net will be opened in new tab. Now you can edit or simulate your net.

Remote repository

Once you are logged in, you can access projects stored in the repository or commit your project. To download project from the repository, select Repository in menu bar and then click on Browse. A new window will be opened and you can browse the repository. Select project to view versions. You can download version from repository by doubleclick on selected version. It is possible to search the repository by entering keyword.

To view a simulation log (called simlog) of a project, open the project in editor, select View Simlog from toolbar. Now you can see history of simulation.

Simulation

A simulation is done on server-side. To simulate petri net, select Run or Simulate from toolbar. To proccess single step, select Step. To proccess full simulation, select Run.

To view a simulation log (called simlog) of a project, open project in editor select View Simlog from toolbar. You can see history of the simulation.

Syntax and operators

You can use these operators in simulation for mode and condition:

OperatorExplanation
+Addition
++Increment
-Substraction
--Decrement
/Division
%Modulus
*Multiplication
==Test for equality
=Assign
&&Logical and
||Logical or
<Less than
<=Less or equal
>Greater than
>=Greater or equal

Example:

Condition: x <= y + 2 || y == 1 && (x % 2 == 0)
Mode: z1 = x * y; z2 = x - y

TIP: You can use C-style assign operators like += or *=. You can use brackets to specify priority as well.

Appearance and configuration

Select Options - Appearance from menu bar. A new window will appear, where you can specify theme for Petri net items and a font to be used. You can edit default login information which will be loaded on application start up.
